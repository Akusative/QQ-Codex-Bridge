# PR 人设与多对话设计

## 1. 名词

- `PR`：人设/角色，不表示 Pull Request。
- 人设条目：分类、人设名称、人设内容。
- 对话窗口：具有独立本地聊天记录、Codex 会话 ID、名称和总结游标的连续会话。
- QQ 账号：作为机器人登录 NapCat 并接入 Bridge 的 QQ；允许未来同时接入多个账号。

## 2. 共享与隔离

全局共享：

- PR 人设库与人设分类。
- 用户基础设定。
- 安全规则与禁止进入记忆的内容类型。

按机器人 QQ 隔离：

- 对话窗口列表与当前窗口。
- 每个窗口绑定的人设；切换窗口时恢复该窗口的人设，切换 QQ 时只显示该 QQ 的窗口绑定关系。
- 原始聊天记录、Codex 会话 ID、总结进度和容量设置。
- 任务状态、待确认请求及自动总结任务。

WebUI 切换 QQ 时显示机器人 QQ 号和昵称；除共享的人设库与用户基础设定外，页面数据整体切换到目标 QQ 的隔离空间。

## 3. 本地目录

```text
workspace\bridge-data\
├─ shared\
│  ├─ personas.json
│  └─ user-base.json
└─ accounts\
   └─ <机器人QQ号>\
      ├─ account.json
      ├─ settings.json
      └─ conversations\
         ├─ index.json
         └─ <conversation-id>.jsonl
```

- `bridge-data` 仅保存在本机，不进入 Git、私有记忆仓库或日志。
- JSONL 逐条保存用户与助手消息；每条记录包含时间、角色、当时使用的人设 ID，不保存 Token 或认证状态。
- 密码、API Key、Token、Cookie、验证码、私钥、身份证信息继续在写入聊天记录前拦截；普通私人内容与剧情内容允许本地保存。

## 4. 人设规则

- PR 内容作为低于系统安全规则的角色背景注入，不得扩大权限、覆盖隐私规则或授权危险操作。
- 切换对话窗口不切换当前人设。
- WebUI 可按分类查看、创建、编辑、选择和停用人设。
- QQ 发送 `/查看人设列表` 时，回复编号、分类与名称列表；发送 `/切换人设 01` 完成切换。
- `/查看当前人设` 只返回当前窗口绑定的人设信息，与查看整个人设列表严格区分；旧命令 `/查看人设` 仅作为列表命令的兼容别名。
- 每条消息记录实际使用的人设，便于同一窗口中途换人设后仍可审计。

## 5. 对话窗口与名称

- 第一次收到普通消息且没有当前窗口时自动建立窗口。
- 未经用户改名的窗口使用首条消息开始的北京时间命名，格式固定为 `年/月/日/时/分/秒`。
- `/查看对话` 返回当前 QQ 的窗口编号与名称；`/切换对话 01` 切换到指定窗口。
- `/修改对话名称 <名称>` 修改当前窗口名称；WebUI 提供同等操作。
- 每个窗口保存自己的人设绑定；切换窗口时同时恢复该窗口的人设，用户可在聊天页下拉框中随时改绑。

## 6. 自动总结与记忆

- 触发条件：切出当前窗口；每天北京时间 00:00；Bridge 启动后发现错过午夜任务。
- 只处理上次总结游标之后的新消息，避免重复总结。
- 摘要只保留稳定的用户细节、偏好、对话习惯、项目习惯、人物关系、事件与规则，不保存逐句复述。
- 涉及私人剧情时使用安全、抽象描述；无法安全概括的片段跳过。
- 摘要必须再次经过凭证、身份信息和敏感格式扫描；不合格条目不进入记忆，也不写入 Git 提交。
- 合格摘要自动写入并提交到既有私人记忆仓库，无需逐条确认；自动条目必须标记来源、QQ 账号空间、对话 ID 与总结时间。
- 更新完成后，只向对应 QQ 发送不含记忆正文的状态事实，再由当前人设生成自然提醒。
- 每三天主动提醒用户检查自动记忆；提醒不自动删除或改写记忆。

## 7. 容量管理

- WebUI 为每个机器人 QQ 单独设置聊天记录上限，并显示已用容量、上限和百分比。
- 默认上限在首次启用时由用户在本机设置页确认，不把磁盘全部空间默认授权给 Bridge。
- 达到 80% 时仅提醒一次；降至 75% 以下后可在再次达到 80% 时重新提醒。
- 提醒只包含容量状态和建议动作，不引用聊天正文；措辞由当前人设组织。
- 达到 100% 时停止接收会产生新本地记录的普通任务，明确要求先清理或提高上限，禁止静默丢弃聊天。
- WebUI 支持按窗口查看占用、导出、删除和提高上限；删除前必须二次确认，并在删除前处理尚未总结的新消息。

## 8. 多 QQ 接入

- 数据键使用 OneBot 事件中的机器人 `self_id`，消息去重键同时包含 `self_id`、用户 ID 和消息 ID。
- WebUI 账号列表来自实际连接的 NapCat 账号；昵称通过受信任的 OneBot 登录信息接口获取，不接受浏览器伪造。
- 每个机器人 QQ 必须有独立的发送通道与连接状态；不得用一个 HTTP 客户端猜测目标账号。
- 第二个 QQ 未完成独立 NapCat HTTP/WS 配置前，只启用单账号数据模型，不伪造“多账号已可用”。

## 9. 首批指令

- `/清空对话`：清空后续 Codex 上下文，但保留本地可见聊天记录。
- `/新对话`：创建新的对话窗口。
- `/查看对话`：列出当前 QQ 的窗口编号与名称。
- `/切换对话 01`：按编号切换当前窗口。
- `/查看当前人设`：只查看当前窗口绑定的人设名称、分类与文档数。
- `/查看人设列表`：列出全部人设编号与名称；`/查看人设` 为兼容别名。
- `/切换人设 01`：按编号为当前窗口切换人设；`00` 表示默认助手。

## 10. WebUI 改版

### 顶部账号区

- 显示当前机器人 QQ 的头像占位、昵称和完整 QQ 号。
- 提供 QQ 下拉切换；切换后重新加载该账号的窗口、当前人设、任务、容量和自动总结状态。
- 明确标注共享内容：人设库与用户基础设定不会随 QQ 切换。

### 对话工作区

- 桌面端左侧显示窗口列表，移动端使用抽屉；支持新建、切换、改名和经二次确认后删除。
- 聊天标题同时显示当前窗口名称、当前人设名称和是否正在使用长期记忆。
- 对话区下方显示可点击的内置指令卡片；点击只填入输入框，不自动发送，防止误操作。
- 切换窗口前显示“正在整理尚未总结内容”；总结完成或安全跳过后再完成切换。
- 未改名窗口直接显示首条消息的北京时间名称。

### 人设库

- 按分类筛选，条目显示名称、启用状态和资料文档数量；内容只在编辑或预览时展开。
- 支持创建、编辑、停用和选择人设；在聊天页切换人设只修改当前窗口的绑定，不切换窗口。
- 聊天标题区并列显示“当前窗口”和“当前人设”下拉框；两个下拉框都列出已有条目，并以“新建一个”作为末项。
- 左侧窗口列表显示每个窗口当前绑定的人设名称，避免把消息发到错误角色上下文。
- 人设编辑器必须提示：内容不能覆盖安全规则，也不得录入密码、Token、Cookie、验证码、私钥或身份证信息。
- 人设采用“核心设定 + 多份资料文档”的 Project 式结构；核心设定可以留空，不再要求把完整人设压进单个文本框。
- 首批文档格式包括 TXT、Markdown、JSON、YAML、CSV、HTML、DOCX 和 PDF，并可在同一个人设中继续增加或移除。
- 上传文档先在内存中提取文字并执行凭证扫描；通过后只保存规范化文本副本，不保存原始二进制文件及其隐藏元数据。
- 运行任务时按当前问题从整组文档中检索相关片段，不把全部资料无条件塞入单次上下文；完整规范化资料仍保留在本地人设库。

### 容量与本地记录

- 设置页按 QQ 显示聊天记录已用容量、上限、百分比和各窗口占用。
- 可调整上限；达到 80% 时显示醒目但不泄露正文的提醒，并同步由当前人设向 QQ 提醒。
- 达到 100% 时禁用发送按钮，提供清理与提高上限入口，不静默丢失消息。

### 自动记忆

- 显示上次自动总结时间、下次北京时间 00:00 任务、待总结消息数量和最近一次结果。
- 提供“立即总结当前窗口”和“检查记忆”入口；立即总结仍执行相同敏感复检。
- 每三天的检查提醒可在设置页看到下次时间，但不能关闭敏感信息复检。

### 响应式布局

- 桌面端：账号栏 + 窗口侧栏 + 主聊天区 + 可切换的人设/状态面板。
- 手机和平板：底部导航保留，账号、窗口和人设分别进入抽屉或独立标签，发送按钮不得被导航遮挡。
- 所有切换操作必须有当前 QQ、窗口和人设的可见反馈，避免把消息发到错误账号或错误上下文。
