# 备选方向：Tailscale 与远程访问

## 当前状态

- 状态：已搁置，可恢复，也可永久放弃。
- 当前实施：WebUI 只保留本机与可信局域网访问，不开放公网入口。
- 搁置原因：Tailscale 官方客户端当前无法通过中国区 App Store 获取，iPhone 与 iPad 的落地条件不足。
- 本文仅保存技术方向与恢复条件，不代表已安装、已开放或已授权公网访问。

## 已确认的可行方向

### Tailscale 私有网络

- Windows、手机、MacBook 与 iPad 加入同一私人 Tailnet 后，可在离家时访问 Bridge。
- 使用 Tailscale Serve 提供仅限 Tailnet 的 HTTPS 地址，不启用公开互联网的 Funnel。
- WebUI 仍保留独立远程密码；Tailscale 是第一层设备准入，WebUI 密码是第二层。
- 桥接主机必须保持开机，NapCat、Bridge 和 Tailscale 必须保持运行。

### 浏览器直接访问的替代方向

- 可进一步评估 Cloudflare Tunnel 与 Access，使移动设备无需安装 VPN 客户端。
- 该方向需要独立域名、云端账号、访问策略和新的隐私评估，不与当前内网实现混合。

### 自有公网入口

- 仅允许采用 HTTPS 反向代理，不得把 `3080` 的明文 HTTP 服务直接映射到公网。
- 非标准高位端口只能减少扫描噪声，不能作为身份验证或安全措施。
- 启用前必须确认公网地址、运营商入站限制、路由器权限、域名或证书方案、限速、锁定和一键关闭入口。

## 不随搁置而改变的安全边界

- 本机免密管理员只认真实回环连接与明确的 localhost Host；任何代理请求都不得继承本机管理员权限。
- 密码只在桥接主机的本机设置页创建或重置，QQ、远程页面、日志、`.env` 和文档不得接收密码正文。
- 密码重置必须撤销全部远程会话；丢失设备时同时撤销网络设备身份与 WebUI 会话。
- 不启用默认密码、万能恢复码、公开 Funnel、裸 HTTP、公网 3080 或通过“冷门端口”替代认证。

## 恢复讨论的触发条件

满足任意一项时可重新评估：

1. 中国区 App Store 可正常获取 Tailscale 官方客户端。
2. 用户已有可长期使用的其他地区 Apple ID，且愿意自行维护设备登录。
3. 用户准备好独立域名并接受 Cloudflare Access 等浏览器方案的云端依赖。
4. 用户明确要求建设自有 HTTPS 公网入口，并具备路由器管理权限和稳定入站条件。

## 恢复时的检查顺序

1. 重新核对官方客户端、套餐和区域可用性。
2. 选择 Tailscale、浏览器访问网关或自有 HTTPS 中的一种，不同时混搭试运行。
3. 先完成测试环境验证，再开放真实 Bridge。
4. 验证本机管理员隔离、远程密码、设备撤销、失败限速、日志最小化和断网回退。
5. 用户从手机网络完成端到端验收后，才把方向从“备选”改为“已实施”。
